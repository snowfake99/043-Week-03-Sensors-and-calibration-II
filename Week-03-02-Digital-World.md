# 3.2 โลกดิจิทัล (Digital World)

โลกดิจิทัลคือโลกที่ทุกสิ่งถูกแทนด้วย **ตัวเลข (Numbers)** ไม่ว่าจะเป็นข้อมูล ภาพ เสียง การเคลื่อนไหว หรือสถานะของระบบ ทุกอย่างถูกแปลงให้อยู่ในรูปแบบที่เครื่องจักรสามารถเก็บ ประมวลผล และส่งต่อได้อย่างแม่นยำและเชื่อถือได้

ในโลกดิจิทัล ข้อมูลไม่ได้มีลักษณะต่อเนื่องเหมือนโลกกายภาพ แต่ถูกแบ่งเป็นหน่วยย่อยที่มีค่าเฉพาะเจาะจง (Discrete Values) ทำให้สามารถจัดการข้อมูลได้อย่างมีประสิทธิภาพและตรวจสอบความถูกต้องได้ง่ายกว่าโลกจริง

### **3.2.1 ตัวเลขในโลกดิจิทัล (Digital Numeric Representations)**

ข้อมูลทุกชนิดในระบบดิจิทัลถูกแทนด้วยตัวเลขประเภทต่าง ๆ เช่น

**Integer (จำนวนเต็ม)** 
	ใช้แทนค่าที่ไม่มีทศนิยม เช่น 0, 1, 100, -50 เหมาะกับการนับจำนวนหรือสถานะที่มีค่าแน่นอน

**Float / Real (จำนวนจริง)** 
	ใช้แทนค่าที่มีทศนิยม เช่น 25.6, 3.14159 เหมาะกับข้อมูลที่ต้องการความละเอียด เช่น อุณหภูมิ, ความสว่าง

 **Boolean (ตรรกะ)** 
	 ใช้แทนสถานะจริง/เท็จ เช่น true/false หรือ 0/1 เหมาะกับการควบคุม เช่น เปิด/ปิด

**Binary (เลขฐานสอง)** 
	รูปแบบข้อมูลพื้นฐานที่สุด เช่น 0b10110101 เป็นภาษาที่เครื่องจักรเข้าใจโดยตรง


### **3.2.2 ลักษณะสำคัญของโลกดิจิทัล (Key Characteristics)**

#### **3.2.1.1 เป็น Discrete (ไม่ต่อเนื่อง)**

ข้อมูลดิจิทัลมีค่าเฉพาะเจาะจง เช่น 0–255, 0–1023, หรือ 0–4095 ตามจำนวนบิตของ ADC ไม่มีค่ากลางระหว่างตัวเลข เช่น ไม่มีค่า 1.234567 ถ้าเป็นระบบ 8-bit ที่รองรับเพียง 0–255

 
 
#### **3.2.2.2 มีความแม่นยำที่กำหนดได้ (Precision)**

ความละเอียดของข้อมูลดิจิทัลขึ้นกับจำนวนบิต เช่น:
- 8-bit → 256 ค่า
- 10-bit → 1024 ค่า
- 12-bit → 4096 ค่า
- 16-bit → 65536 ค่า
ยิ่งบิตมาก → ยิ่งละเอียด → ยิ่งใกล้เคียงโลกจริง

#### **3.2.2.3 เก็บ ส่ง ประมวลผล และทำซ้ำได้โดยไม่สูญเสียข้อมูล**

ข้อมูลดิจิทัลสามารถ:
- เก็บในหน่วยความจำโดยไม่เสื่อมสภาพ
- ส่งผ่านเครือข่ายโดยไม่ผิดเพี้ยน
- คัดลอกซ้ำกี่ครั้งก็ยังเหมือนเดิม
- ประมวลผลได้อย่างแม่นยำและรวดเร็ว

 

#### **3.2.2.4 รองรับการตรวจสอบและแก้ไขข้อผิดพลาด (Error Detection & Correction)**

โลกดิจิทัลสามารถตรวจสอบความถูกต้องของข้อมูลได้ เช่น:
- **Parity bit**
- **Checksum**
- **CRC (Cyclic Redundancy Check)**
- **ECC (Error Correction Code)**

สิ่งเหล่านี้ทำให้ข้อมูลดิจิทัลมีความน่าเชื่อถือสูงมาก แม้จะส่งผ่านเครือข่ายที่มีสัญญาณรบกวน

### **3.2.3 โลกดิจิทัลไม่ได้หมายถึงแค่ 0 และ 1**

แม้ข้อมูลพื้นฐานจะเป็นเลขฐานสอง แต่ระบบดิจิทัลประกอบด้วยองค์ประกอบที่ซับซ้อนกว่านั้นมาก

#### **3.2.3.1 Data Storage (การเก็บข้อมูล)**

- Flash
- EEPROM
- SD Card

ใช้เก็บข้อมูลระยะยาว เช่น ค่าคอนฟิก, Log, ข้อมูลเซนเซอร์

#### **3.2.3.2 Data Processing (การประมวลผล)**

- MCU (Microcontroller)
- CPU (Central Processing Unit)
- FPGA (Field Programmable Gate Array)

ใช้ประมวลผลข้อมูลตามตรรกะที่กำหนด เช่น ควบคุมมอเตอร์, วิเคราะห์สัญญาณ

#### **3.2.3.3 Data Communication (การสื่อสาร)**

- UART
- I2C
- SPI
- Wi-Fi
- BLE
- Ethernet

ใช้ส่งข้อมูลระหว่างอุปกรณ์หรือขึ้นระบบคลาวด์

#### **3.2.3.4 Data Integrity (ความถูกต้องของข้อมูล)**

- CRC
- Checksum
- Parity bit

ใช้ตรวจสอบว่าข้อมูลที่ส่งหรือเก็บยังถูกต้องหรือไม่

## **3.2.4 การนำเสนอข้อมูลในโลกดิจิทัล (Digital Data Representation)**

#### **3.2.4.1 ต้องมีหน่วยวัดประกอบเสมอ (Metadata)**

ตัวเลขเพียงอย่างเดียวไม่เพียงพอ เช่น:
- 25.6 → อุณหภูมิ? ความสว่าง? ความเร็วลม?
- 1023 → ค่า ADC? ความดัน? ระดับน้ำ?

ต้องมี metadata เช่น:
- หน่วยวัด (°C, lux, Pa)
- เวลา (timestamp)
- ตำแหน่ง (location)
- แหล่งข้อมูล (sensor ID)

#### **3.2.4.2 การนำเสนอตัวเลขอย่างเดียวอาจทำให้ข้อมูลขาดความสมบูรณ์**

ตัวอย่างเช่น
```
25.6
```

ถ้าไม่มีหน่วย → ไม่มีความหมาย ถ้าไม่มีเวลา → ไม่รู้ว่าเกิดเมื่อไร ถ้าไม่มี sensor ID → ไม่รู้ว่ามาจากอุปกรณ์ใด

#### **3.2.4.3 การเก็บข้อมูลในรูปตัวเลขต้องมีโครงสร้าง**

เช่น
- ตารางเวลา (Time-series table)
- Log file
- JSON
- CSV
- Database

ตัวอย่างตารางกระแสไฟฟ้า

| Time (ms) | Current (A) |
| --------- | ----------- |
| 0         | 0.12        |
| 10        | 0.15        |
| 20        | 0.18        |

#### **3.2.4.4 การนำเสนอข้อมูลกายภาพในโลกดิจิทัลช่วยให้สร้าง Digital Twin**

Digital Twin คือแบบจำลองดิจิทัลของวัตถุจริง เช่น:
- มอเตอร์
- อาคาร
- ระบบผลิต
- เครื่องจักร

ข้อมูลจากเซนเซอร์ → ดิจิทัล → แบบจำลอง → วิเคราะห์ → ควบคุม
